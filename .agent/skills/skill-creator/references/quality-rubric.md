# Skill Quality Rubric

Scoring guide for evaluating skill quality. Use this to review new skills or audit existing ones.

---

## Scoring Scale

| Score | Level | Description |
|-------|-------|-------------|
| 5 | **Exceptional** | Best-in-class, could be an industry reference |
| 4 | **Strong** | Comprehensive and effective, minor gaps only |
| 3 | **Adequate** | Functional but needs improvement |
| 2 | **Weak** | Missing key elements, limited utility |
| 1 | **Poor** | Fundamentally flawed, needs rewrite |

---

## Evaluation Categories

### 1. Structure (20 points max)

| Criteria | 5 pts | 3 pts | 1 pt |
|----------|-------|-------|------|
| **YAML Frontmatter** | Complete with triggers & references | Present but incomplete | Missing or malformed |
| **Section Organization** | Logical flow, easy to navigate | Mostly organized | Confusing or chaotic |
| **Heading Hierarchy** | Perfect H1‚ÜíH2‚ÜíH3 structure | Minor issues | No hierarchy |
| **Cross-references** | Links to all related skills | Some links | No cross-references |

**Score: ___ / 20**

---

### 2. Content Quality (30 points max)

| Criteria | 5 pts | 3 pts | 1 pt |
|----------|-------|-------|------|
| **Expertise Definition** | Clear, specific role established | Generic expertise | No role definition |
| **Principles** | Foundational, memorable, actionable | Present but generic | Missing or vague |
| **Framework** | Comprehensive methodology | Basic steps | No clear process |
| **Examples** | Multiple, concrete, varied | Few examples | No examples |
| **Checklists** | Thorough, categorized | Basic lists | No validation |
| **Output Format** | Specific deliverables defined | General guidance | No output spec |

**Score: ___ / 30**

---

### 3. Actionability (25 points max)

| Criteria | 5 pts | 3 pts | 1 pt |
|----------|-------|-------|------|
| **Specificity** | Exact steps, no ambiguity | Mostly specific | Vague guidance |
| **Completeness** | Full task coverage | Most scenarios | Major gaps |
| **Practicality** | Ready to implement now | Needs adaptation | Theoretical only |
| **Context Gathering** | Thorough initial assessment | Basic questions | No discovery |
| **Error Prevention** | Addresses common mistakes | Some warnings | No pitfall guidance |

**Score: ___ / 25**

---

### 4. Technical Quality (for dev skills) (15 points max)

| Criteria | 5 pts | 3 pts | 1 pt |
|----------|-------|-------|------|
| **Code Examples** | Production-ready, well-commented | Basic snippets | No code or broken |
| **Security** | Comprehensive security guidance | Basic mentions | No security |
| **Performance** | Optimization considerations | Some awareness | Ignored |

**Score: ___ / 15** (or N/A for non-dev skills)

---

### 5. Usability (10 points max)

| Criteria | 5 pts | 3 pts | 1 pt |
|----------|-------|-------|------|
| **Readability** | Scannable, excellent formatting | Readable but dense | Wall of text |
| **Trigger Accuracy** | Activates exactly when needed | Sometimes misfires | Wrong triggers |

**Score: ___ / 10**

## Quick Quality Checks

### Red Flags (Immediate Concern)

- [ ] No YAML frontmatter
- [ ] Missing Initial Assessment section
- [ ] No code examples in technical skill
- [ ] No checklists
- [ ] Generic copy-paste content
- [ ] Broken internal references
- [ ] No Output Format section

### Green Flags (Quality Indicators)

- [x] Clear role expertise statement
- [x] Numbered, prioritized steps
- [x] Tables for comparison data
- [x] Before/After examples
- [x] Common mistakes section
- [x] Related skills properly linked
- [x] Task-specific questions
